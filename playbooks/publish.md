# 发布流程 Playbook

## ⚠️ 必读文件（每次触发前强制读取）

```bash
cat ./config/platform-rules.md
```
> 里面有数据验证过的：标题公式、发布时间、粉丝画像、内容类型适配、封面策略。
> 原始数据在 `配置/platform-analytics.md`，但**规则都在速查文件里**，不需要读 600 行。

## 全平台文案（触发：录好了/全平台文案）

1. **先读 `配置/发布规则速查.md`**（标题公式、发布时间、禁忌）
2. 生成各平台文案（参考 `视频/发布方案/` 下历史模板排版）
3. 质量校验（必须严格执行下方【全平台质量校验规格】）
4. **如果佳蔓对生成的文案提出了微调反馈，必须第一时间将反馈固化写入 `配置/发布规则速查.md` 或本 Playbook 中，做到经验自动传承**。
5. `pbcopy` 各平台文案
6. 更新 `<YOUR_WORKSPACE>/publish-tracker.md`

## 发完了（触发：发完了/发好了/上线了）

1. 问佳蔓哪些平台标题/时间/标签有改动
2. 记录实际发布数据 → 追加到 `publish-tracker.md`
3. 如有改动 → 同步更新 `配置/发布规则速查.md` 中的偏好
4. 下次“全平台文案”自动应用最新偏好

## 长文发布（触发：发长文）

markdown → 公众号（原文复制，不用markdown版）+ Substack + 小红书图文 + X + IG

长文发布时自动生成信息图配合小红书图文：
```bash
notebooklm generate infographic "English description, clay 3D style, white background, soft lavender purple" --orientation portrait --wait
```

## 全平台质量校验规格（强制包含全部 8 个视频平台全矩阵）

| 平台 | 校验排版与格式硬性要求 |
|------|--------|
| **YouTube** | 标题≤100字，描述前三行抛核心亮点，必备：时间戳、导流官网 jiamanjemma.com，标签总长≤500字符 |
| **TikTok** | 标题极简（单句极速抓眼球），配 3-5 个高热度英文或中英混合标签（#AI #Productivity 等），符合海外强视觉、快节奏阅读习惯 |
| **Instagram** | 重视排版（用空行/标点分段），首段直接给利益点或共鸣。必须在正文提及 "Link in bio" 引导点击。底部堆叠 15-20 个以上中英热门 #标签 |
| **X (Twitter)** | 极度精简克制，开头第一句必须直接交代核心成就/价值点/暴论。总长度控制在 2-3 句话。带 2-3 个核心精准标签。绝不写口水话 |
| **视频号** | 产出文案严禁使用【标题】、【简介】、【标签】等小标题结构。必须将开头到标签部分合并写成一个连贯的大段落直接一键粘贴用于“视频描述”。但是，必须在纯净内容之后**单独输出一行短标题（字数建议6-16个字符）**，对应后台底部的独立输入框。<br>标签必须带#并包含 #<YOUR_IP_NAME> |
| **B站** | 标题≤25字。<br>文案里不要杂乱加标签，统一在**【标题】下方直接输出逗号隔开的无井号纯净【标签】段落**（专用于复制进后台输入框）。<br>**严禁任何“求三连”、“求关注”话术**。<br>原“动态”改名为“视频简介”；并在下方强制附带一段“粉丝动态（一键转发模式）”话术引导老粉观看 |
| **小红书** | 标题≤20字，必备emoji，**严禁感叹号开头**。为了完播率，正文前两句必须设极其吸引人的悬念/痛点。<br>字数300-500，底部附带不超过 10 个 #标签 |
| **抖音** | 标题≤30字，跟小红书一样前两秒文案生死时速埋强悬念。<br>结尾加入清晰的“合集归属”，标签带#且不超过 5 个 |
