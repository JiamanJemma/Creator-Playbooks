# 发布流程 Playbook

## ⚠️ 必读文件（每次触发前强制读取）

```bash
cat ./config/platform-rules.md
```
> 里面有数据验证过的：标题公式、发布时间、粉丝画像、内容类型适配、封面策略。
> 原始数据在 `配置/platform-analytics.md`，但**规则都在速查文件里**，不需要读 600 行。

## 全平台文案（触发：录好了/全平台文案）

1. **先读 `配置/发布规则速查.md`**（标题公式、发布时间、禁忌）
2. 生成各平台文案（参考 `视频/发布方案/` 下历史模板排版）
3. 质量校验（必须严格执行下方【全平台质量校验规格】）
4. **如果佳蔓对生成的文案提出了微调反馈，必须第一时间将反馈固化写入 `配置/发布规则速查.md` 或本 Playbook 中，做到经验自动传承**。
5. `pbcopy` 各平台文案
6. 更新 `<YOUR_WORKSPACE>/publish-tracker.md`

## 发完了（触发：发完了/发好了/上线了）

1. 问佳蔓哪些平台标题/时间/标签有改动
2. 记录实际发布数据 → 追加到 `publish-tracker.md`
3. 如有改动 → 同步更新 `配置/发布规则速查.md` 中的偏好
4. 下次“全平台文案”自动应用最新偏好

## 全平台质量校验规格（强制包含全部主要视频平台）

| 平台 | 校验排版与格式硬性要求 |
|------|--------|
| **YouTube** | 标题≤100字，描述前三行抛核心亮点，必备：时间戳、导流官网。标签【严禁带#号，只用逗号分隔】。必须错开国内发布，选定次日上午 08:30 (北美通勤) |
| **TikTok** | 根本没有单列的标题栏，必须将大标题、正文、标签 100% 融合在一个 4000 字符内的大框中。必须错峰发布。|
| **Instagram/X** | 极度精简，直接放核心图文+视频直达链接+Github开源链接作为顶层流量漏斗吸引点击。 |
| **视频号** | 第一行到最后一行全为正文（含带 # 标签），但必须在代码块或文本最后单独拆出一行 6-16 字的短标题，专门用来填底部独立的极窄输入框。必须勾选原创声明。 |
| **B站** | 标题≤25字物理限制。文案里的标签严禁带 #，最后留一块纯净带回车的标签堆叠区。 |
| **小红书** | 标题≤20字物理限制，正文前两句设悬念。必提醒句末加话题并在后台加入合集与勾选原创。 |
| **抖音** | 标题≤30字，区分“工作标题”和“简介”，标签带#≤5个。发文助手必须勾选自主声明。 |
